# State Update Rules

## Update after meaningful work
- completed work
- verification result
- next work
- risks or notes

## Update on close
- summarize completed task
- record verification
- record next starting point
- record unresolved risks if any

## New project registration
1. Add registry entry.
2. Create a state file from the workspace template.
3. Record path, aliases, run command, verify command, deploy method, key docs.

## Writing style
- Keep state files concise.
- Prefer operational facts over narrative.
- Keep current goal, recent completed work, and next work easy to scan.
