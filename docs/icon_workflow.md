# 아이콘 작업 공용 규칙

## 목적

워크스페이스 내 프로젝트에서 아이콘이 필요할 때, 매번 다른 방식으로 만들지 않고 일관된 기준으로 작업하기 위한 공용 규칙입니다.

## 기본 원칙

- 아이콘 작업이 필요하면 먼저 `Font Awesome`으로 해결 가능한지 검토합니다.
- 단순 도형, 상태, 액션, 툴바, 메뉴 아이콘은 이미지 생성보다 `Font Awesome` 같은 아이콘 폰트를 우선합니다.
- 웹 화면에서는 가능하면 아이콘 폰트를 직접 사용합니다.
- 크롬 확장프로그램처럼 비트맵 파일이 필요한 경우에는 `Font Awesome` 기반으로 PNG 세트를 만듭니다.

## 권장 우선순위

1. `Font Awesome` 아이콘 직접 사용
2. `Font Awesome` 기반 PNG export
3. 기존 프로젝트 자산 재사용
4. 위 방법으로 해결되지 않을 때만 별도 이미지/그래픽 제작

## 적용 기준

### 웹앱 / 웹페이지

- 우선 `<i class="fas ...">` 또는 동등한 아이콘 폰트 방식으로 검토합니다.
- CSS로 색상, 크기, 여백, 배지를 조정합니다.
- 동일 프로젝트 안에서는 같은 아이콘 계열을 유지합니다.

### 크롬 확장프로그램

- `manifest.json`에 연결할 아이콘은 별도 PNG 파일이 필요합니다.
- 먼저 `Font Awesome` 기준 아이콘을 정합니다.
- 이후 원형 배지, 배경색, 여백 규칙을 정해 `16/32/48/128` PNG 세트로 만듭니다.

권장 규격:

- `icon-16.png`
- `icon-32.png`
- `icon-48.png`
- `icon-128.png`

## 작업 체크리스트

- 이 아이콘은 `Font Awesome`으로 표현 가능한가
- 웹에서 직접 폰트 아이콘으로 써도 되는가
- 확장프로그램이라 PNG 세트가 필요한가
- 기존 프로젝트의 아이콘 톤과 맞는가
- 같은 의미의 아이콘을 다른 스타일로 중복 생성하고 있지는 않은가

## 기록 원칙

- 아이콘 작업을 했다면 변경 이력이나 상태 파일에 아래 중 하나를 남깁니다.
  - `Font Awesome 직접 사용`
  - `Font Awesome 기반 PNG 아이콘 세트 생성`
  - `기존 자산 재사용`

## 예시

### 예시 1. 웹 메뉴 아이콘

- 관리자 메뉴, 설정 메뉴, 경고 아이콘
- 처리 방식: `Font Awesome` 직접 사용

### 예시 2. 크롬 확장프로그램 아이콘

- 브라우저 툴바 아이콘
- 처리 방식: `Font Awesome` 기준 형태 선택 후 PNG export

## 메모

- 앞으로 아이콘 요청이 들어오면 먼저 이 문서를 기준으로 판단합니다.
- 이미지 생성은 마지막 수단으로 남깁니다.
